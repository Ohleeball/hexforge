# Design Pillars

These principles guide all design decisions. When in doubt, return here.

---

## 1. Every Decision Has Weight

Nothing is reversible for free. Walls don't repair. Sold buildings require a new blueprint. Tier upgrades destroy two of the three buildings that created them. Players should feel the consequences of their choices throughout the run, not reset them.

## 2. The Two Loops Feed Each Other

The action loop (move, fight, capture) and the building loop (plan, place, upgrade) must be causally connected in both directions. Fighting generates gold and blueprints. Buildings improve fighting capability. Neither loop is a minigame bolted onto the other.

## 3. Tension Over Comfort

The game should never feel safe. When you're defending, you're not building. When you're building, you're not defending. When you're out capturing a zone, your base is taking damage. Mechanics should amplify this tension, not resolve it.

## 4. Spatial Thinking Is a Skill

Base layout is a puzzle. Adjacency bonuses reward planning ahead. The third building of a type determines where the tier 2 lands. Players who think spatially should be rewarded over players who place randomly.

## 5. Depth Through Systems, Not Content

Replayability comes from emergent combinations of systems — building synergies, economy decisions, risk/reward of distance — not from content volume. A small set of well-designed systems beats a large set of shallow ones.

## 6. Readable at a Glance

The player should be able to assess their situation — wall health, gold, incoming enemies, base layout — without reading tooltips mid-fight. Visual clarity is not a polish concern, it is a design constraint.

## 7. Prototype First

If a mechanic cannot be prototyped simply, it should be simplified until it can. Complexity is earned by proven fun, not assumed in advance.
